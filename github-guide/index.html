1️⃣ 깃허브 5대 핵심 개념
💾
저장소 (Repository)
"우리의 공용 클라우드 폴더"
프로젝트의 모든 파일과 과거 기록이 저장되는 공간입니다. 타임머신 기능이 있어 언제든 과거로 되돌릴 수 있습니다.
📸
커밋 (Commit)
"작업 중간 저장 & 인증샷"
"내가 이 부분을 이렇게 바꿨어"라고 메모를 남기며 사진을 찍어두는 행위입니다.
🌌
브랜치 (Branch)
"평행 우주 만들기"
메인 프로젝트를 망치지 않기 위해, 현재 상태를 복사해서 나만의 작업 공간을 만드는 것입니다.
📤📥
푸시 & 풀 (Push & Pull)
"올리기와 내려받기"
Push: 내 작업을 서버로 밀어 올립니다. Pull: 최신 내용을 내 컴퓨터로 당겨옵니다.
🔍
풀 리퀘스트 (PR)
"결재 올리기"
내 브랜치에서 작업을 마친 뒤, "메인에 합쳐도 될까요?"라고 검토를 요청하는 과정입니다.
✅ 따라해보기 체크리스트
2️⃣ 매일 반복할 '4단계 루틴'
1
작업 전 최신화
작업 시작 전 Pull을 하여 최신 버전을 다운받아 옵니다.
2
브랜치 생성
Main 브랜치가 아닌, 새로운 브랜치를 만들어 이동합니다.
3
작업 & 기록
업무를 수정하고 메모를 남긴 뒤, Push하여 저장합니다.
4
PR 요청
작업 완료 후 Pull Request로 검토를 요청합니다.
🎓 핵심 개념 퀴즈
Q. 브랜치를 만드는 이유는 무엇일까요?
A. 메인 프로젝트를 직접 수정하기 위해
B. 메인을 보호하면서 나만의 작업 공간을 만들기 위해
C. 파일 업로드 속도를 높이기 위해
