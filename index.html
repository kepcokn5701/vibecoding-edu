CHAPTER 01
바이브코딩(Vibe Coding)이란?
프로그래밍 언어 대신 자연어로 AI에게 지시하여 소프트웨어를 만드는 새로운 패러다임입니다.
💬

기존 코딩 vs 바이브코딩

한눈에 비교하기

▾
❌ 기존 방식 — 직접 코딩
import pandas as pd
df = pd.read_excel('전력사용량.xlsx')
df['peak'] = df['usage'].rolling(24).max()
fig = px.line(df, x='time', y='peak')
fig.update_layout(title='피크전력분석')
# ... 수십~수백 줄 코드 필요
VS
✅ 바이브코딩 — AI에게 지시
"지난 1년간 월별 피크전력 사용량을
분석해서 그래프로 보여주고,
이상치가 있으면 빨간색으로
표시해줘. 엑셀 파일은 여기 있어."
💡
핵심 차이: 기존 코딩은 '어떻게(How)' 만들지를 코드로 지시하지만, 바이브코딩은 '무엇(What)'을 원하는지만 자연어로 설명합니다. AI가 'How'를 알아서 처리합니다.
🔑

핵심 개념 3가지

자연어 지시, AI 코드 생성, 반복 개선

▾
💬

자연어 프롬프팅

프로그래밍 언어 대신 일상 언어(한국어)로 원하는 결과를 설명합니다.

🤖

AI 코드 생성

Claude, ChatGPT 등이 지시를 이해하고 실행 가능한 코드를 자동 작성합니다.

🔄

반복적 개선

"색상 바꿔줘", "그래프 추가해" 처럼 대화로 계속 수정·발전시킬 수 있습니다.

⚡

전력산업에서 왜 필요한가?

디지털 전환 시대의 필수 역량

▾
⏱️

반복 업무 자동화

엑셀 정리, 보고서 작성, 점검 기록 관리를 자동화합니다.

📊

데이터 분석

전력 수급, 설비 이력, 고장 패턴을 직접 분석·시각화합니다.

💡

아이디어 즉시 구현

IT부서 의뢰 없이 필요한 도구를 10분 안에 만듭니다.

💰

비용 절감

외주 개발 비용 없이 간단한 업무 도구를 직접 제작합니다.

CHAPTER 02
도구 준비하기
바이브코딩에 사용하는 AI 도구들을 알아보고, 자신에게 맞는 도구를 선택합니다.
🧰

추천 AI 도구 비교

숙련도와 용도에 따라 선택하세요

▾
🟣

Claude (Anthropic)

한국어 우수, 파일 생성·다운로드 지원. 입문자 최추천

🟢

ChatGPT (OpenAI)

범용적 활용, Code Interpreter로 데이터 분석 가능.

🔵

Cursor IDE

AI 내장 코드 에디터. 프로젝트 단위 개발에 최적.

🟡

v0.dev (Vercel)

프롬프트로 웹 UI 즉시 생성. 바로 배포 가능.

🎯
첫 시작 추천 경로: Claude에서 HTML 파일 생성 → 다운로드 → GitHub 업로드 → Vercel 배포. 가장 간단하고 빠른 방법입니다.
📝

좋은 프롬프트 작성법

AI에게 잘 지시하는 것이 핵심 역량입니다

▾

바이브코딩의 품질은 프롬프트(지시문) 품질에 달려 있습니다. 다음 공식을 기억하세요:

✏️
좋은 프롬프트 = 누가 + 무엇을 + 왜 + 구체적 요구사항
❌ 나쁜 프롬프트
"변압기 앱 만들어줘"
VS
✅ 좋은 프롬프트
"현장 기사가 모바일에서 쓸 수 있는
변압기 점검 체크리스트 웹앱을 만들어줘.
점검 항목: 절연저항, 유중가스분석, 온도측정.
각 항목마다 정상/주의/이상 선택 가능.
이상 선택 시 사진 첨부 기능.
점검 완료 시 PDF 리포트 출력.
HTML 파일 하나로 만들어줘."

전력산업 활용 사례 모음

✅

✅
데이터 기반 의사결정 지원, 전력 수급 안정성 향상

안전사고 예방, 절차 준수율 100% 달성

✅

CHAPTER 04
GitHub에 코드 올리기
만든 파일을 GitHub에 올리면 안전하게 보관되고, Vercel 배포와 자동으로 연결됩니다.
📦

GitHub 저장소 만들기

코드의 구글 드라이브를 만듭니다

▾

GitHub은 코드를 저장·관리하는 온라인 서비스입니다. 코드의 구글 드라이브라고 생각하세요.

  • ✓github.com 접속 → "Sign up" → 무료 계정 생성
  • ✓우측 상단 "+" → "New repository" 클릭
  • ✓이름 입력 (예: power-equipment-checker)
  • ✓"Public" 선택, "Add a README file" 체크
  • ✓"Create repository" 클릭 → 완료!
💡 GitHub 협업 자세히 알아보기 (비개발자용)
📤

파일 업로드하기

초보: 드래그&드롭 / 중급: Git 명령어

▾
  • ✓저장소 페이지에서 "Add file" → "Upload files"
  • ✓AI가 만들어준 파일을 드래그 & 드롭
  • ✓"Commit changes" 클릭 → 끝!
⚠️
중요: 메인 HTML 파일 이름을 index.html로 해야 Vercel이 홈페이지로 인식합니다.
Terminal
# 저장소 복제
$ git clone https://github.com/내아이디/my-app.git
$ cd my-app
# AI가 만든 파일을 이 폴더에 넣고
$ git add .
$ git commit -m "feat: 전력설비 점검 앱"
$ git push
✅ 업로드 완료!
📁 my-power-app/
📄 index.html ← 메인 (필수)
📄 README.md ← 프로젝트 설명
💡 GitHub 협업 자세히 알아보기 (비개발자용)

CHAPTER 05
Vercel로 전 세계에 배포하기
GitHub 저장소를 연결하면 자동으로 웹사이트가 만들어집니다. 무료입니다.
🚀

3분 배포 가이드

클릭 몇 번이면 웹사이트가 생깁니다

▾
  • ✓vercel.com 접속 → "Start Deploying"
  • ✓"Continue with GitHub" → GitHub 로그인
  • ✓"Add New... → Project" 클릭
  • ✓내 저장소를 찾아 "Import" 클릭
  • ✓설정은 그대로 두고 "Deploy" 클릭
  • ✓1~2분 후... 🎉 배포 완료!
🎊
